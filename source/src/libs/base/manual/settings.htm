Einstellungen

1. URL zur Startseite

Der URL zur Startseite wird benötigt um Sie immer an die richtige Position leiten zu können. Dadurch wird gewährleistet dass die Seite auch bei Frames ordnungsgemäß dargestellt wird und nicht nur ein einzelner Frame angezeigt wird.

 

2. Textdarstellung

Umwandlung von Zeilenumbrüchen

TIP: Es ist zu empfehlen den speziellen und kostenlosen Editor (FCKEDITOR) für das CMS zu verwenden. Diesen erhalten Sie auf Anfrage kostenlos bzw. steht zum Download auf der Internetseite des CMS zur Verfügung. Sollten Sie den erweiterten Text-Editor (FCKEDITOR) verwenden, wird die Umwandlung von Zeilenumbrüchen kaum noch benötigt. Somit müssen Sie sich nicht unbedingt mit dieser Funktion befassen. Da es sich bei dem Editor um ein OpenSource Produkt eines Drittanbieters handelt, kann keine Garantie oder Gewährleistung in jeder Hinsicht gegeben werden. Weitere Informationen finden Sie auf www.fckeditor.net

Wenn Sie einen Menüpunkt hinzufügen werden alle Zeilenumbrüche (Return/Enter Taste) dargestellt, so wie Sie Ihren Text eingegeben haben.

Wenn Sie aber spezielle HTML Formatierungs Befehle (TAGs) eingeben kann es vorkommen, dass dadurch die spezielle Formatierung verzerrt oder gar nicht dargestellt wird. In diesem Fall muss die Umwandlung von Zeilenumbrüchen abgeschaltet werden (Option: NEIN). Dies wirkt sich allerdings auf alle Menüpunkte aus. Bei der Option JA wird der Zeilenumbruch IMMER umgewandelt.

Damit Sie HTML TAGs sowie normalen Text verwenden können und dieser einwandfrei dargestellt wird, gibt es die Option AUTO. Hierbei durchsucht Beeco Ihren Text des Menüpunktes und entscheidet ob die Umwandlung des Zeilenumbruches stattfindet oder nicht.

Um leichte Formatierungen mit HTML Befehlen (TAGs) dennoch vornehmen zu können, die unter umständen der Darstellung nicht schaden, gibt es den Tag Filter. In dem Tag-Filter sind HTML Formatierungsbefehle mit Komma getrennt eingetragen. Wenn Beeco im AUTO Modus einen dieser TAGs in Ihrem Text findet, wird dennoch der Zeilenumbruch umgewandelt. Findet Beeco einen anderen TAG, wird der Zeilenumbruch nicht umgewandelt.

 

3. Template Override (Experts only ;-)

Für gewöhnlich verwendet Beeco immer das Standard Template (Design-Vorlage). Dies lässt sich über die URL auch nicht verändern.

Wenn Sie eine Internetseite machen möchten die z.B. Frames nutzt und somit in jedem Frame ein anderer Inhalt dargestellt werden soll, benötigen Sie unterschiedliche Templates (Design-Vorlagen), für jedes Frame Fenster.

Wenn Sie Beeco auffordern möchten ein anderes Template zu verwenden, müssen Sie zuerst "Template-Override" aktivieren, damit Ihre Aufforderung von Beeco akzeptiert wird.

Anschließend müssen in Ihrem Frameset beim Aufruf der Beeco Startseite (index.php) folgende Parameter anhängen: ?style=NewTemplateName

"NewTemplateName steht hierbei für den Namen Ihres Templates in dem Template/User Ordner. Somit akzeptiert Beeco Ihre Aufforderung und lädt das vorgegeben Template Damit auch Ihre Menüpunkte das richtige Template laden (für das passende Frame Fenster), müssen Sie an den Platzhalter %URL% in den ITEMS Abschnitten ebenfalls ?style=NewTemplateName anhängen.

Weiteres zum Aufbau der Templates und dessen Platzhalter finden Sie in der Template-Beschreibung in der Manual.

 

4. Zugangsdaten

Für die optimale Sicherheit der Beeco Verwaltung wird ein Benutzername und ein Kennwort benötigt.
Nur mit diesen Zugangsdaten können Sie sich in der Verwaltung anmelden und Ihre Internetseite mit Inhalt füllen. Sollten Sie das Kennwort oder den Benutzernamen vergessen haben, wenden Sie sich an jemanden mit ausreichenden MySQL Datenbank Rechten um Ihren Benutzer auslesen zu können. Im Falle des Passwortes fordern Sie bitte bei dem Author Hendrik Reimers (hendrik.reimers@web.de) Unterstützung an.

 

5. Weitere Konfigurationen (Experts only ;-)

Bei der Installation des CMS haben Sie eine Datei bearbeitet im Ordner "includes" und dort unter "settings". Diese Datei hieß "mysql.settings.php" und wird benötigt um die Verbindung zu der Datenbank zu ermöglichen und die Daten von Ihnen zu speichern bzw. zu verwalten.

Diese Datei ist allerdings nicht die einzigste zusätzliche Konfigurationsdatei. Es gibt noch weitere die Einfluss auf das CMS haben. Diese liegen ebenfalls unter "/user/settings/base/", beziehungsweise unter "/user/settings/plugins/PLUGIN_NAME/". Zum Beispiel finden Sie in den angegebenen Ordner(n) häufig die Datei "other.settings.php".

Die Möglichkeiten der Konfiguration die Sie in diesen Dateien vornehmen können, haben häufig extrem große Auswirkungen auf das CMS bzw. das jeweilige Plugin und sollten daher nur von einem Experten verändert. Eine Beschreibung zu den jeweiligen Optionen die Sie dort verändern können, finden Sie in der jeweiligen Datei.
